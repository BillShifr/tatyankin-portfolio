# КРИТИЧЕСКАЯ ПРОБЛЕМА: Конфликт DNS записей для www

## Проблема

Для `www.tatyankin-portfolio.ru` одновременно существуют:
- ✅ CNAME запись → `billshifr.github.io` (правильно)
- ❌ A-записи (IPv4) → `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153` (конфликт!)
- ❌ AAAA-записи (IPv6) → `2606:50c0:8001::153`, `2606:50c0:8000::153`, `2606:50c0:8002::153`, `2606:50c0:8003::153` (конфликт!)

**Это недопустимо!** Нельзя иметь одновременно CNAME и A/AAAA записи для одного домена.

## Решение

### В REG.RU → Управление DNS → для домена `tatyankin-portfolio.ru`:

1. **Удалите ВСЕ A-записи для `www`** (если они есть)
2. **Удалите ВСЕ AAAA-записи для `www`** (если они есть)
3. **Оставьте ТОЛЬКО CNAME запись для `www`**:
   - Имя: `www`
   - Тип: `CNAME`
   - Значение: `billshifr.github.io`
   - TTL: `3600`

### Для основного домена (apex):

**Оставьте ТОЛЬКО 4 A-записи** (без CNAME для apex):
- `@` → `185.199.108.153`
- `@` → `185.199.109.153`
- `@` → `185.199.110.153`
- `@` → `185.199.111.153`

## После исправления

1. Подождите 15-30 минут для распространения DNS
2. Проверьте DNS:
   ```bash
   dig @8.8.8.8 www.tatyankin-portfolio.ru CNAME +short
   # Должно вернуть: billshifr.github.io.
   
   dig @8.8.8.8 www.tatyankin-portfolio.ru A +short
   # Должно вернуть ПУСТО (нет A-записей)
   
   dig @8.8.8.8 www.tatyankin-portfolio.ru AAAA +short
   # Должно вернуть ПУСТО (нет AAAA-записей)
   ```
3. В GitHub Pages → Settings → Pages → Custom domain:
   - Удалите домен (если он там есть)
   - Подождите 1 минуту
   - Добавьте снова: `tatyankin-portfolio.ru`
   - Нажмите "Save"
   - Подождите проверки DNS (может занять до 5 минут)

## Важно

- Для apex domain (`tatyankin-portfolio.ru`) - только A-записи
- Для www (`www.tatyankin-portfolio.ru`) - только CNAME
- Никаких конфликтов между CNAME и A/AAAA записями!
